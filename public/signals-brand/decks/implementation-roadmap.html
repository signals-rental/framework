SIG-IR03Phase 0Rev A

Phase 0: Architectural Foundations

Every Phase 0 week saves three weeks in Phase 7. These are the load-bearing walls of the entire system — permissions, field metadata, event dispatch, and API conventions that every future feature will depend on.

Getting these contracts right means every subsequent phase slots in cleanly. Getting them wrong means cascading rewrites that compound with every new domain.

This phase produces no visible features for end users, but without it, nothing else works. The engines, registries, and service contracts established here are the foundation everything builds upon.

SIG-IR04Phase 1Rev A

Phase 1: People & Places

Members are the universal entity. Contacts, organisations, venues, and system users all live in one table differentiated by membership_type. This eliminates entity proliferation and enables powerful cross-entity features.

A single member can be a contact at one organisation, a user of the system, and the venue liaison for a location — all without data duplication.

This phase also establishes localisation foundations: countries, currencies, timezones, and formatting rules that every financial and scheduling feature will use.

SIG-IR05Phase 2Rev A

Phase 2: Product Catalogue

Product groups (hierarchical), items (products + services), stock levels (serialised + bulk), accessories (bundling), kits (serialised components).

This phase also introduces the financial engines that underpin all pricing. Multi-currency support with lossless integer minor-unit storage, tax zone resolution, and the composable rate engine.

Every financial calculation in the system — quotes, invoices, reports — flows through these engines. They must be correct before Phase 3 can build on them.

Rate Engine

Product Catalogue
Groups, items, stock levels, accessories, kits

Pricing flows upward: products define rates, rates produce prices, the tax engine handles currency and tax.

SIG-IR08Phase 4Rev A

Phase 4: Billing & Financial

Invoices, payments, credit notes, deposit and pro-forma invoicing. Complete quote-to-cash. A rental company could actually run their business on this.

The four-stage pricing pipeline ensures deterministic, auditable price calculation. Each stage transforms the price independently, producing a clear trail from list price to final charge.

All money is stored as integers in minor units (pence, cents) using brick/money value objects. No floats, no rounding errors. API responses show decimal strings for CRMS compatibility.

SIG-IR09Phase 5Rev A

Phase 5: Infrastructure Services

The four services that make the system production-ready. Without these, it is a prototype. With them, companies can actually switch.

01
Notification Engine
Multi-channel (email, SMS, in-app, push). Three-layer resolution: type defaults, tenant settings, member preferences. Template-driven with Blade rendering.
02
Audit & Event Projection
Complete mutation history. Before/after snapshots for every write. Temporal queries for point-in-time reconstruction. Configurable retention with scheduled pruning.
03
Import / Export Engine
SQLite-staged migration pipeline. Pre-built CRMS field mappings for one-click migration. Dry-run validation. Rollback capability. Progress tracking via cache.
04
Document Template Engine
Blade templates per store. PDF rendering via DomPDF. Field browser for template authors. Public sharing via signed URLs. Sequential document numbering.
SIG-IR10Phase 6Rev A

Phase 6: Compliance & Fleet

Inspections (PAT testing, safety certificates), certifications with expiry tracking, quarantine management for damaged or under-repair assets, stock checks, and fleet vehicle management.

The scanning abstraction layer unifies barcode, QR code, RFID, and manual entry into a single pipeline. Three layers: input adapters normalise raw scans, the resolution engine identifies the entity, and context handlers execute the appropriate action.

Assets follow a lifecycle from acquisition through active service, inspection, potential quarantine for repair, and eventual retirement or disposal.

SIG-IR12Phase 8Rev A

Phase 8: External Interfaces

The full REST API surface, scheduling and resource allocation engine, CRM features (activities, discussions, comments, tags), and global search.

The API is CRMS-compatible by design: identical field names, Ransack-compatible query syntax, matching response shapes. Existing integrations and reports built for Current RMS work with minimal changes.

The scheduling engine handles crew, vehicles, facilities, and equipment as a unified resource model with conflict detection and optimisation suggestions.
